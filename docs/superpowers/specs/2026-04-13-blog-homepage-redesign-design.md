# Blog Homepage Redesign Design

## Goal
将当前偏模板化的博客首页重构为“极简科技”风格的个人品牌首页，视觉效果参考大品牌官网的简洁、高级、克制与秩序感，同时保留博客首页对文章内容的快速触达能力。

## Scope
本次设计仅覆盖首页 `index.html` 的视觉与信息层级重构，不包含文章详情页、归档页、标签页的同步改版，也不引入新的前端框架或构建工具。

## Current State
当前首页基于 Bootstrap 默认组件拼装，存在以下问题：

- 导航、Hero、文章卡片、页脚之间的品牌语言不统一。
- 首页首屏只有站点名，缺少能体现个人定位的品牌表达。
- 文章列表使用密集的小卡片网格，信息完整但高级感不足，模板痕迹明显。
- 辅助元素较多，视觉重心分散，留白与节奏不够克制。
- 页脚社交图标偏多，降低了整体的“官网化”精致感。

## Design Direction
整体采用“极简科技 + 品牌化博客”的设计方向：

- 以浅色背景、柔和层次、充足留白建立官网质感。
- 通过大字号标题、克制副文案和精简导航强化品牌表达。
- 将首页从“文章卡片墙”转为“品牌首屏 + 精选内容列表”的结构。
- 保留博客的内容入口效率，避免为了视觉而牺牲可读性。

## Visual Language

### Color
- 背景以暖白或冷白为主，不使用大面积纯白刺眼底色。
- 主文字使用深灰而非纯黑，提升质感。
- 强调色使用低饱和蓝灰或银灰，只在按钮、标签、细节高光中出现。
- 阴影保持轻微，仅用于分层，不制造厚重“卡片感”。

### Typography
- 首页主标题使用明显更大的字号与更宽松的行高，作为首屏视觉核心。
- 二级标题与区块标题保持统一体系，减少默认 Bootstrap 字体层级感。
- 正文与辅助信息严格区分：说明文案清晰但简短，日期和 tag 降低对比度。
- 整页依靠 typography 和 spacing 形成高级感，而不是依赖复杂装饰。

### Surface and Motion
- 使用极轻的边框、半透明底色和模糊效果塑造 floating navigation 与内容容器。
- 背景使用轻量 radial gradient、柔和光斑或极弱纹理提升质感。
- hover 动效只允许轻微位移、阴影变化或背景过渡，避免夸张动画。

## Information Architecture
首页拆为五个主要区块：

1. Floating Header
2. Hero Section
3. Featured Content Section
4. Recent Posts Section
5. Minimal Footer

### 1. Floating Header
目标是让导航更像品牌官网，而不是传统博客菜单。

- 左侧保留站点名 `evabd21`。
- 右侧只保留高价值入口，控制在 2 到 3 个：`About`、`Archives`、`Github`。
- 其中 `About` 在实现阶段直接复用现有 `/getting-start` 链接，不新增新页面路由。
- 导航容器使用半透明浅色背景、轻边框和圆角，视觉上悬浮于页面顶部。
- 保留现有主题切换入口，但在视觉上弱化，不打断主导航秩序。

### 2. Hero Section
Hero 是本次改版的品牌中心。

- 主标题不只显示用户名，而是以“个人品牌 + 方向说明”的方式呈现。
- 副标题用一句话说明博客内容方向，例如安全研究、漏洞分析、Java Security。
- 提供一个主按钮和一个次按钮：
  - 主按钮用于进入文章列表或滚动到内容区。
  - 次按钮用于外链，如 Github。
- Hero 区使用更大的垂直留白，确保首屏有充分呼吸感。
- 背景不使用复杂图片，避免破坏极简科技气质。

### 3. Featured Content Section
首页第一篇内容作为“精选文章”展示，而不是与其他文章完全等权。

- 从现有文章列表中抽取最新或最适合展示的一篇作为 featured post。
- Featured post 默认使用首页文章列表中按日期最新的一篇，避免新增人工维护逻辑。
- Featured 区采用更大的容器、更高层级标题、更完整的摘要呈现。
- 该区块作为 Hero 和内容列表之间的过渡，建立内容权重与节奏。

### 4. Recent Posts Section
普通文章列表从模板卡片墙改为更高级、更克制的内容模块。

- 放弃当前密集的小网格卡片，改为单列或更宽松的双列结构。
- 每个条目包含标题、日期、简短摘要和标签，但层级更清晰。
- 内容容器弱化“卡片”边界，强调排版与间距而非重边框。
- 移动端保持单列，桌面端优先保证阅读节奏而不是追求一屏塞更多内容。

### 5. Minimal Footer
页脚需要明显收敛。

- 保留版权信息和必要链接。
- 删除或收敛当前大量社交图标，只保留真正有价值的品牌出口。
- 页脚的视觉存在感应低于正文区，作为克制收尾而不是次级导航中心。

## Content Strategy
- 首页文案不追求长篇自述，只保留一句明确、克制、专业的描述。
- 文章列表优先突出标题质量与主题标签，避免辅助信息抢占视觉注意力。
- 如果现有首页缺少摘要字段，可先使用现有内容结构进行轻量展示，不为本次改版额外引入复杂数据逻辑。

## Responsive Strategy
- 桌面端重点表现留白、层级和 Hero 质感。
- 平板端保持相同信息顺序，但减少横向分栏。
- 移动端改为单列堆叠，优先保证标题、按钮、文章入口清晰可点。
- 不允许因视觉改版降低移动端的阅读效率与点击可用性。

## Technical Constraints
- 继续使用现有静态 HTML + Bootstrap 资源，不新增构建链路。
- 样式改动优先通过首页内联样式或已有样式资源实现，避免引入过多额外依赖。
- 保持现有链接结构与文章入口不变，避免影响静态站点导航。
- 不修改文章数据源，仅调整首页的 HTML 结构与样式表达。

## Risks and Mitigations

### Risk 1: 只改样式不改结构，仍然有强模板感
Mitigation:
- 必须同时调整 Hero、列表结构、导航与页脚，而不是只换颜色。

### Risk 2: 过度官网化导致博客内容入口变弱
Mitigation:
- Hero 下方直接接 featured post 与 recent posts，确保用户一屏半内可以看到内容入口。

### Risk 3: 视觉细节过多导致后续维护困难
Mitigation:
- 效果集中在首页公共区块，避免对每篇文章卡片写复杂特例。

### Risk 4: 移动端因为大留白和大字号失衡
Mitigation:
- 在实现中为移动端单独控制字号、间距与按钮尺寸，优先可读性。

## Acceptance Criteria
- 首页第一眼观感明显摆脱默认 Bootstrap 模板风格。
- 首屏具备明确品牌表达，而不只是站点名占位。
- 导航、Hero、内容区、页脚形成统一的极简科技视觉语言。
- 文章入口依然清晰，首页不会因为“高级感”而难用。
- 桌面端与移动端都保持简洁、清楚、自然的阅读节奏。

## Out of Scope
- 文章详情页、标签页、归档页同步视觉升级。
- 内容管理方式调整。
- 引入 JavaScript 复杂交互、动画库或全新 UI 框架。
